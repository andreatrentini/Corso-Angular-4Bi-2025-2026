Bindings

Per bindings si intendono gli strumenti che ci sonsento di collegare e quindi rendere dinamici i template HTML con attributi e metodi della classe (typescript) del componente.

String interpolation

Con string interpolation visulizzo nel template il valore di una "variabile" typescript.

ATTENZIONE: le variabili che possono essere visualizzate sono solo ed esclusivamente attributi della classe.

In un progetto zoneless per garantire che la vista venga aggiornata se combia il valore della variabile DEVO usare i signals e non varibili tradizionali.

Cosa sono i signals? Sono funzioni wrapper (funzioni contenitori, cioe avvolgono, gestiscono un valore che può essere di qualsiasi tipo: string, number, array, oggetti.
Un signal per la modifica dei valori mette a disposizione dei metodi, se vengono usati, il nuovo valore del signal verrà visualizzato nel browser.

Quali sono gli step?

  1. Dichiarare un signal in typescript ed assegnargli un valore
  2. Inserire una chiamata alla funzione (signal è una funzione wrapper) fra doppie parentesi graffe.

Event binding

Consente di eseguire codice typescript al verificarsi di un evento

Come si procede

  2. dal nome dell'evento togliete on iniziale e inserite quello che resta fra parentesi tonde
  3. dovete aggiungere un metodo alla classe con il codice da eseguire

Property binding

Con property binding posso impostare il valore di un attributo di un tag html in modo dinamico usando variabili/signals typescript.

Per utilizzare property binding racchiudo il nome dell'attributo fra parentesi quadre, e fra le virgolette inserisco la "variabile/signal" che contiene il valore dell'attributo.

Style binding

Attraverso style binding è possibile impostare uno o più stili di un elemento html in modo dinamico. Si realizza utilizzando la sintassi [style.nomestiledaimpostare]="espressione typescript che determina il valore."

Class binding

Attraverso class binding è possibile impostare l'attributo class di un elemento html. Si realizza utilizzando la sintassi [class]="elenco delle classi da aggiungere", oppure [class.nomeclasse]="espressione typescript che restiuisce vero/falso".
